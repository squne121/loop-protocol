# Security Policy

## Reporting

脆弱性や公開すべきでない情報を見つけた場合は、公開 Issue に書かず、GitHub の private security advisory か maintainer への直接連絡で報告してください。

## Scope

- ソースコードの脆弱性
- ワークフローや GitHub Actions の権限設定
- 誤って公開された機密情報や権利情報

## Notes

- `assets/` と `LICENSES/` はコードとは別に確認します。
- AI エージェント運用に関する問題も security issue として扱って構いません。
